# Swipe · Páginas de venta y OTOs ganadoras (copy real)

Banco de **páginas que venden** (con precio y checkout), distinto del opt-in de registro
(`swipe-landings.md`). Aquí vive la arquitectura del **funnel ascendente de PBS**: un workshop de
entrada barato → upsells (OTO) inmediatos.

> **De dónde sale.** Páginas reales de PBS marcadas como ganadoras (sin métricas por página en la
> fuente). El funnel es un **tripwire → OTO**:
> ```
> Workshop $11 (Supervisión Comercial con IA)  →  OTO1 $197 (Diagnóstico Fábrica de Ingresos)
>                                               →  OTO2 $47  (Entrenamiento "Pipeline Predecible")
> ```
> **Alta relevancia FAP:** el OTO1 ($197) es, en esencia, **el diagnóstico de FAP en miniatura**
> (6 pilares, Revenue Architect, sesión 1:1, Scorecard, Índice de Dependencia del Fundador).

## 🧠 Patrones observados

**Esqueleto de página de venta (tripwire) PBS:**
countdown → eyebrow → **H1 = mecanismo + resultado + plazo** → CTA con precio → prueba social
(logos + Nº directivos) → diagrama/modelo → **dolores ("para ti si…")** → **tablas de resultados
cuantificados** → testimonios con nombre → bio de Jorge (autoridad) → demo del producto/entregable
→ stack "lo que recibes" → **garantía** → justificación de precio → FAQ → cierre.

**Esqueleto de OTO (upsell inmediato):**
interrupción ("aún NO está completa") + barra de progreso → H1 de deseo → reencuadre ("no es
venta, es X") → **demo del entregable antes de comprarlo** → cómo funciona (pasos) → ancla de
precio ("vale ~$X → hoy $Y") → **doble CTA sí/no** con el "no" que reafirma lo que se pierde →
nota de honestidad.

**Recursos transversales (los que dan confianza):**
- **Ancla de precio** siempre ($2.000 → $197; $477 → $47; "$11 es lo más bajo").
- **Honestidad como diferenciador:** "no hay contadores falsos ni cupos inventados" — construye
  confianza justo donde el mercado suele mentir.
- **Garantía sin riesgo** (7 días, "y te quedas con el material").
- **Prueba cuantificada** (tablas antes/después con % y $) por encima de adjetivos.
- **Escasez real** (cupos en vivo, "oferta solo en esta página"), no fabricada.
- **Reencuadre de la oferta** ("no es una llamada de ventas / no es un curso normal").

## Notas abiertas

- Estas páginas están marcadas como ganadoras pero **sin métricas por página** (CVR de checkout,
  take-rate de cada OTO). Si Jorge tiene el take-rate de los OTO, se rankea el funnel con dato duro.
- El **OTO1 ($197 diagnóstico)** es prioridad para el cerebro de FAP: es el molde de la pieza de
  **diagnóstico/aplicación** de FAP. Cuando se construya `fap-landings` o una skill de diagnóstico,
  esta es la referencia.
- Falta por cargar: la **página de ventas del programa pago principal** (SPG / el propio FAP) y la
  **página de gracias / confirmación** post-checkout.
